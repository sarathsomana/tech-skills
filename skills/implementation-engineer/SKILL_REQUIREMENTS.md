# Skill Requirements: Implementation Engineer

## 1. Primary Objective
The `Implementation-Engineer` skill acts as a general feature builder. It ingests architectural blueprints (such as those generated by the `first-principles-design` skill) and executes the phased coding, testing, and wiring of business logic required to implement features. It bridges the gap between high-rigor architectural design and code execution.

## 2. Context & "Ground Truth"
- **Context:** Belongs to Phase 2: Core Engineering Skills.
- **Ground Truth:** The skill relies on:
  1. Architectural blueprints and design artifacts (e.g., from `first-principles-design`).
  2. The existing project codebase context and file structure.
  3. Established coding standards and practices within the repository.

## 3. Core Problems Solved
- Haphazard or unguided implementation of complex features.
- Drift between architectural design intent and actual code implementation.
- Missing integration and wiring of discrete components.

## 4. Key Execution Phases (Proposed)
1. **PLANNING (The Strategist):** Ingest the architectural blueprint and current codebase state to formulate a concrete, step-by-step coding and integration plan.
2. **IMPLEMENTATION (The Builder):** Execute the plan by writing the business logic, creating necessary files, and implementing the core feature logic.
3. **WIRING (The Integrator):** Connect the newly built components to the rest of the application (e.g., routing, state management, API endpoints).
4. **VERIFICATION (The Tester):** Validate the implementation against the original architectural blueprint to ensure no state drift and verify that all requirements are met.
5. **SIGN_OFF (The Reviewer):** Obtain final user sign-off and document the completed implementation.
